Install HomeAssistant Core in Termux

Vladut Stefan
2021-02-27

After playing with HomeAssistant on different devices, I’ve seen a post about installing HomeAssistant in Termux on an old Android device. (source).

Don’t grab a coffee yet. You will have time while waiting at Step 5. (Or maybe grab one and think about the 2nd one you’ll drink later)

I didn’t want to invest a lot of time in unlocking the bootloader and installing a custom OS (Lineage) on my Samsung Galaxy Tab 4 (SM-T530) like the original article sugested.
The only gain I’ve imagined I would have by doing this is bloatware removal, and maybe a newer android kernel version.

Well, lo and behold Termux actually gives a shit about how old your Android device is and will bite you in the ass while you try to install diferent stuff on it.

But, after a night of searchings, HomeAssistant Core in Termux on old Android devices is doable or a tale of how to link math libs and other libs in clang on termux given you can find hints in error messages.

1: Get Termux from wherever you like.

  • PlayStore
  • FDroid
  • etc

1.1 (Optional) Install sshd or dropbear

Termux Docs

I like to make my life complicated but this time I actually had 2 neurons free and they thought I should install something to ssh into Termux. But you can skip this and type every command from the tablet. (You sick fuck!)

I’ve decided to go with dropbear. No specific reason why.

2: Install python pip etc.

Yes. That simple. Termux decided to bind -dev and helper packages inside default packages name. So if you need python-dev, you just install python.

3: Create a virtual env and get in.

This is optional from a technical/functional perspective but for the sake of cleaness, do it!

This should print back the name of your virtual env name before the $

When you want to exit just type:

4: Install clang Pillow and libjpeg-turbo.

Some HomeAssistant dependencies will fail to build without this.
I tried gcc but again, some packages complain.
I’m sure there is a way to fix those but meh, it was 12AM on a Friday after a week of work.

(If you want to go down the gcc rabbit hole, here is a starting point!)

Pillow is (I think) a image library that HomeAssistant uses.
Don’t take my word for it. I saw the “image” keyword in the error and searched the internet.
Pillow depends on libjpeg.

The LDFLAGS is the key here. (https://wiki.termux.com/wiki/Image_Editors)
-lm links the math library
-L links the system libraries..

Don’t ask me why math is not a system library. Maybe the system had a stupid math teacher in Highschool, maybe it had a crush on math and it broke it’s heart. Or maybe beause of this?

5: Install homeassistant (Finally).

While in the virtual env

Go get/make a coffee. Don’t forget to link the coffee, water and electricity libraries and don’t use gcc. At least not this time.

6: Make this run on boot.
