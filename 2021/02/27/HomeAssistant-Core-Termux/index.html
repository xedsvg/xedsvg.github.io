Install HomeAssistant Core in Termux

Vladut Stefan
2021-02-27

After playing with HomeAssistant on different devices, i’ve seen a post about installing HomeAssistant in Termux on an old Android device. (source)

Don’t grab a coffee yet. You will have time while waiting at Step 5.

I didn’t want to invest a lot of time in unlocking the bootloader and installing a custom OS (Lineage). The only gain I’ve imagined I would have by doing this is bloatware removal, and maybe a newer android kernel version.

Well, lo and behold Termux actually gives a shit about how old your Android device is and will bite you in the ass while you try to install diferent stuff on it.

But, after a night of searchings, HomeAssistant Core in Termux on old Android devices is doable.

Step 1: Get Termux from wherever you like.

  • PlayStore
  • FDroid
  • etc

(My installed version is 0.83 at the moment of writing this)

Step 1.1 (Optional) Install sshd or dropbear https://wiki.termux.com/wiki/Remote_Access

I like to make my life complicated but this time I actually had 2 neurons free and they tought i should install something to ssh into Termux. But you can skip this and type every command from the tablet.

I’ve decided to go with dropbear. No specific reason why.

Create a password for the user you are gonna use.

Find what user you are gonna use:

Open dropbear in background

Connect to your tablet from another computer (ssh port is 8022)

Step 2: Install python pip etc

Yes. That simple. Termux decided to bind -dev and helper packages inside default packages name. So if you need python-dev, you just install python.

Step 3: Create a virtual env and get in
I will not tell you ‘this is optional’. This is optional from a technical/functional perspective but for the sake of cleaness, do it!

This should print back the name of your virtual env name before the $

When you want to exit just type

Step 4: Install clang and Pillow
Some HomeAssistant dependencies will fail to build without this.
I tried gcc but again, some packages complain. I’m sure there is a way to fix those but meh, it was 12AM on a friday after week of work.
(If you want to go down the gcc rabbit hole, here is a starting point https://github.com/its-pointless/gcc_termux)

https://wiki.termux.com/wiki/Image_Editors
Pillow is a image library that I think HomeAssistant uses. Don’t take my word for it. I saw the “image” keyword in the error and searched the internet.

Step 4.1 I’M HERE BECAUSE I HAVE ERRORS, I DID NOT FOLLOW THE STEPS AND I THINK THIS WILL HELP

Clean your .cache folder.
rm ~/.cache

Clean your virtual env folder (this will delete every instaled package inside the virtual env)
python -m virtualenv –clean hass

Continue with steps from point 4.

Step 5: Install homeassistant (Finally)

While in the virtual env

pip install homeassistant

Go get/make a coffee. Don’t forget to link the coffee, water and electricity libraries and don’t use gcc. At least not this time.
